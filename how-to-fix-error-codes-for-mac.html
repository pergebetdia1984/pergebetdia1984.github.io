What is Error Code-43?

Error Code-43 is a type of error arises in Mac OS X 10.2 and Mac OS X EI Capitan. Whenever this error occurs, the MacBook Screen shows a dialogue as under:

“The operation can’t be completed because one or more required items can’t be found. (Error Code-43)”

When does the Error Code-43 occur in Mac?

Error Code-43 occurs when the Mac User attempts to delete/move some files to Trash or move files from one folder to other folder. However, the Error Code-43 does not occur every time, but only when there’s some problem with one or more files you want to move/delete on your Mac PC.

Causes of Mac Error Code-43

Like every action has a reaction, every error occurs due to some reasons. So we should know the Causes of Error Code-43, only then we can find best solution to fix it. There is no single cause for Error Code-43, it arises by various causes. According to our research, Mac Error Code-43 occurs because of Illegal Characters, Incompletely Downloaded Files (Corrupted Files), Locked or Protected Files/Folders, Active File i.e. File in Use, No Shared Point and File from Windows. Here we will explain each cause of Error Code-43 in brief.

[I] Illegal Characters:

When you face Error Code-43 on Mac, you should check the file(s) and folder(s) you tried to move or delete. If a file name contains an Illegal Character then it may be the cause of Error Code-43. The Illegal Characters include File Separators, Non-alphabetical & Nonnumerical Symbols, Punctuation Marks, Brackets, Operators, Parentheses, Quotation Marks and White Space Characters. Some examples of Illegal Characters are . , [ ] { } ( ) ! ; ” ‘ * ? < > |, : (colon), / (forward-slash), (backward-slash),¢, ™, $, ®.

In addition to above mentioned Illegal Characters, a File must not have a name longer than 31 characters. If a file name on your Mac has any of these Illegal Characters, you should rename the file(s) and try again to move/delete them. If the Error Code-43 is wholly caused by Illegal Characters then the Error will not occur after renaming such files. It means you won’t face the Error Code-43 after you remove Illegal Characters from file name.

[II] Incompletely Downloaded Files or Corrupt Files:

It is not possible to move or delete a file which is corrupted or incompletely downloaded. Therefore, you should check if the file you are trying to move/delete is completed downloaded or not. So you should first complete the downloading process and then try to delete or move that file to trash/other folder.

[III] Locked or Protected Files/Folders:

If a file or folder is locked or protected than you cannot delete/move them. This can happen because you are logged in as a guest on MacBook and Only Admin has the right to move or delete that file. Now you have to login as Admin and Unlock the File/Folder to be able to delete/move them and remove the Error Code-43.

[IV] Active File i.e. File in Use:

When a File is Open on System, you cannot delete or move it. The Error Code-43 may appear because of the File being Active/in Use. In such situation, you should stop using that file i.e. check in which App the file is open, Close that App and try again to move/delete that file.

[V] No Shared Point:

The Error Code-43 may appear because the file is saved as No Shared Point. It means the Finder App cannot access that file and therefore you cannot delete or move it.

[VI] File from Windows:

Mac System cannot read a file from Windows. When you transfer a file from Windows PC to Mac PC, the Mac OS can’t read the file directly. And this issue may cause the Error Code-43 on your MacBook.

How to Fix Mac Error Code-43

Error Code-43 is a very small issue happening in Mac PC System. There are some simplest methods to fix Mac Error Code-43 as under:

  1. Force Quit Finder
  2. Reset PRAM or NVRAM
  3. Check Disk Utility
  4. Unlock Folder
  5. Delete Locked Files
  6. Remove Protected File Using RM Command
  7. Clean Windows File with Dot_clean Command

Now we will explain all the above given methods in detail.

01. Force Quit Finder

This is the easiest and risk-free method to fix Error Code-43 on MacBook. Your Mac System is showing Error Code-43, because Finder perhaps there is bug which is preventing Finder to delete or move the file. So if you Force Quit Finder and Relaunch Finder, it will fix the Error Code-43 on your MacBook.

Steps to Force Quit Finder on MacBook:

  1. Open Apple Menu from the MacBook Desktop.
  2. Navigate to the option Force Quit Finder and select it.
  3. Select Finder in Force Quit Applications list.
  4. Click on Relaunch button to proceed.

After Successful Force Quit and Relaunch Finder, try to move or delete that specific file to know if Error Code-43 is fixed. When this method doesn’t work, you should try Resetting PRAM/NVRAM.

02. Reset PRAM or NVRAM

Parameter Random Access Memory (PRAM) or Non-Volatile Random Access Memory (NVRAM) plays an important role in MacBook System. The PRAM or NVRAM is a small amount of memory stored on every MacBook and Mac PC. This memory contains information required for computer before the Mac OS is loaded in System. If you Reset PRAM or NVRAM on your MacBook, it will fix the Error Code-43 issue. Furthermore, Resetting PRAM/NVRAM is Not harmful.

Please follow the steps mentioned below to Reset PRAM or NVRAM on Mac:

  1. Close all applications (if any) and Shut Down your MacBook normally.
  2. After the MacBook is completely turned off, check the four keys Command, Option (Alt), P and R.
  3. Press the Power button to start your MacBook and immediately press those four keys simultaneously i.e. Command, Option (Alt), P, R.
  4. Hold the four keys for 20 Seconds or until you hear Startup Sound three times.
  5. Release the keys and let your MacBook PC Start.

Check the Error Code-43 is fixed otherwise you should go ahead with next method.

03. Check Disk Utility

Sometimes, the Mac System shows Error Code-43 because you have tried to move or delete a Missing or Corrupt File. You can repair such corrupt file(s) with Dist Utility option as described below:

  1. Close all Apps and Open Apple Menu on your MacBook.
  2. Select Restart option and let the Mac System turn off.
  3. You should hold the “Command+R” keys when the MacBook Restarts.
  4. Hold the keys until the Apple logo appears on screen.
  5. Choose Disk Utility and click on Continue.
  6. Select the Disk in which the file, you tried to move/delete, is located.
  7. Click on First Aid and Continue.
  8. Wait till the Check Disk Utility Process gets over.

After completion of Disk Utility Checking, you may see one of the following reports:

Report I: “Overlapped extent allocation” – It means there are some files which occupies same spot and one or more files of them are corrupted. So you need to go to DamagedFiles folder and Repair such files. If those files can’t be repaired, you should delete them.

Report II: “The underlying task reported failure.” – It means the Disk Utility Check Process failed. You should select First Aid and again run the check utility process. If it fails again then backup your data and format/erase that disk.

Report III: “Your Disk/Drive is about to Fail.” – This is an alarming warning for you. Just go to nearby Apple MacBook Service Center and request for the Replacement of Hard Drive (Disk).

04. Unlock Folder

Locked Files and Folders cannot be deleted with normal delete option on Mac. If you want to delete the file(s) which is locked, you should unlock it first and then try to delete it. In case there are more than one files or folder, you can Unlock the Folder from the easiest steps given below:

Steps to Unlock the Folder on Mac:

  1. Select the file/folder and right click to see available options.
  2. Click on “Get Info” to open its information.
  3. Navigate to “Sharing & Permissions” head.
  4. Change Permissions to “Read & Write” for All Users.
  5. Your selected file or folder has been successfully Unlocked.

After the Folder is Unlocked, try to move or delete the files/folders. If you are still not able to Delete that file then try next method “Delete Locked Files”.

05. Delete Locked Files

When you attempt to delete Locked Files, the Mac System can’t do it and shows Error Code-43. Sometimes, you can move Locked Files to Trash, but can’t Delete them permanently from Trash Folder. You can Delete Locked Files from Trash by following below steps:

  1. Open Applications and select Utilities.
  2. Select Terminal App and wait till it is launched.
  3. Type the Command as “chflags -R nouchg” in Terminal and press Enter key.
  4. Provide your Admin Password and again press Enter key.
  5. Drag and Drop Locked/All Files from Trash to Terminal window.
  6. Your Trash will be Emptied and all Locked Files will be permanently Deleted.

06. Force Delete Protected File Using “RM” Command

Protected Files are really tough to move or delete on MacBook. When you select Delete File (Move to Trash) option, the Mac System fails to complete the task. Here we have a simple solution for Protected Files, you can Delete them through following procedure:

  1. Go to Applications, Select Utilities and Open Terminal App.
  2. Type the Command in Terminal as “RM(space)Protected File Location Path.”
  3. Please leave one space after RM and drag & drop the protected file in Terminal.
  4. Now Press Enter key to proceed.
  5. The Protected File has been Removed/Deleted from your MacBook.

07. Clean Windows File with “Dot_clean” Command

As we know that Windows File doesn’t support in Mac and the Mac System can’t read the file transferred from Windows. If there is a file from Windows in your MacBook then you should Clean the Windows File with Dot_Clean Command to get rid of Error Code-43. You need to copy the location of Windows File before you start the procedure. Kindly follow the steps given below:

  1. Open Applications and Select Utilities.
  2. Click on Terminal App to launch it.
  3. Type the Command in Terminal as “Dot_clean” and leave one space.
  4. Then paste the location of Windows File right after space.
  5. Finally click Enter Key to clean the file.
  6. Now you can delete and move that file anywhere in your MacBook.

If Error Code-43 is Not Fixed even after applying all the methods, your MacBook PC may have hardware problem. You should visit nearby Apple MacBook Service Centre and give your MacBook for Repair.
